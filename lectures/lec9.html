Signals

Asychronous notification of an event
  • Process A sends signal to Process B
  • OS interrupts Process B
  • If Process B registered a handler, it handles
  • Otherwise default handler handles

Important Signals

  • SIGINT -- ctrl-c
  • SIGILL -- illegal instruction
  • SIGKILL -- kill signal
  • SIGSTOP -- pauses a process
  • SIGCONT -- resumes a process
  • SIGUSR1 / SIGUSR2

Signals and Threads

Don't mix well:
  • Handling thread
  • Notifying other threads
  • Handling interruption
  • Accidental deadlocking

Pipes

  • Communication channel between two processes
  • Supports write, read, close
  • Bidirectional or unidirectional
  • Half or full duplex
  • Relationship between processes
  • Local or network
  • Linux / POSIX -- FIFO

Unix Pipes


int pipefd[2];
pid_t cpid;
char buf;

pipe(pipefd);
if ((cpid = fork()) == 0)
  close(pipefd[1]);          /* Close unused write end */
  while (read(pipefd[0], &buf, 1) > 0)
    write(STDOUT_FILENO, &buf, 1);

  write(STDOUT_FILENO, "\n", 1);
  close(pipefd[0]);
} else {            /* Parent writes argv[1] to pipe */
  close(pipefd[0]);          /* Close unused read end */
  write(pipefd[1], "hello\0", 6);
  close(pipefd[1]);          /* Reader will see EOF */
  wait(NULL);                /* Wait for child */
}

Semaphores

  • Recall sychronization lecture
  • Resource count
  • P - sem_wait - take
  • V - sem_post - return
  • P and V are atomic operations

Semaphores


void p(semaphore *s) {
  lock(s->control);
  if (s->count == 0) {
    queue(s);
  }
  s->count--;
  unlock(s->control);
}

void v(semaphore *s) {
  lock(s->control);
  s->count++;
  signal(s);
  unlock(s->control);
}
